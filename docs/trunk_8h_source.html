340 /** Allocate a new connection for the trunk
341  *
342  * The trunk code only interacts with underlying connections via the connection API.
343  * As a result the trunk API is shielded from the implementation details of opening
344  * and closing connections.
345  *
346  * When creating new connections, this callback is used to allocate and configure
347  * a new #fr_connection_t, this #fr_connection_t and the fr_connection API is how the
348  * trunk signals the underlying connection that it should start, reconnect, and halt (stop).
349  *
350  * The trunk must be informed when the underlying connection is readable, and,
351  * if `always_writable == false`, when the connection is writable.
352  *
353  * When the connection is readable, a read I/O handler installed by the init()
354  * callback of the #fr_connection_t must either:
355  *
356  * - If there's no underlying I/O library, call `fr_trunk_connection_signal_readable(tconn)`
357  * immediately, relying on the trunk demux callback to perform decoding and demuxing.
358  * - If there is an underlying I/O library, feed any incoming data to that library and
359  * then call #fr_trunk_connection_signal_readable if the underlying I/O library
360  * indicates complete responses are ready for processing.
361  *
362  * When the connection is writable a write I/O handler installed by the open() callback
363  * of the #fr_connection_t must either:
364  *
365  * - If `always_writable == true` - Inform the underlying I/O library that the connection
366  * is writable. The trunk API does not need to be informed as it will immediately pass
367  * through any enqueued requests to the I/O library.
368  * - If `always_writable == false` and there's an underlying I/O library,
369  * call `fr_trunk_connection_signal_writable(tconn)` to allow the trunk mux callback
370  * to pass requests to the underlying I/O library and (optionally) signal the I/O library
371  * that the connection is writable.
372  * - If `always_writable == false` and there's no underlying I/O library,
373  * call `fr_trunk_connection_signal_writable(tconn)` to allow the trunk mux callback
374  * to encode and write requests to a socket.
375  *
376  * @param[in] tconn The trunk connection this connection will be bound to.
377  * Should be used as the context for any #fr_connection_t
378  * allocated.
379  * @param[in] el The event list to use for I/O and timer events.
380  * @param[in] conf Configuration of the #fr_connection_t.
381  * @param[in] log_prefix What to prefix connection log messages with.
382  * @param[in] uctx User context data passed to #fr_trunk_alloc.
383  * @return
384  * - A new fr_connection_t on success (should be in the halted state - the default).
385  * - NULL on error.
386  */
387 typedef fr_connection_t *(*fr_trunk_connection_alloc_t)(fr_trunk_connection_t *tconn, fr_event_list_t *el,
388  fr_connection_conf_t const *conf,
389  char const *log_prefix, void *uctx);

414 /** Multiplex one or more requests into a single connection
415  *
416  * This callback should:
417  *
418  * - Pop one or more requests from the trunk connection's pending queue using
419  * #fr_trunk_connection_pop_request.
420  * - Serialize the protocol request data contained within the trunk request's (treq's)
421  * pctx, writing it to the provided #fr_connection_t (or underlying connection handle).
422  * - Insert the provided treq
423  * into a tracking structure associated with the #fr_connection_t or uctx.
424  * This tracking structure will be used later in the trunk demux callback to match
425  * protocol requests with protocol responses.
426  *
427  * If working at the socket level and a write on a file descriptor indicates
428  * less data was written than was needed, the trunk API client should track the
429  * amount of data written in the protocol request (preq), and should call
430  * `fr_trunk_request_signal_partial(treq)`.
431  * #fr_trunk_request_signal_partial will move the request out of the pending
432  * queue, and store it in the partial slot of the trunk connection.
433  * The next time #fr_trunk_connection_pop_request is called, the partially written
434  * treq will be returned first. The API client should continue writing the partially
435  * written request to the socket.
436  *
437  * After calling #fr_trunk_request_signal_partial this callback *MUST NOT*
438  * call #fr_trunk_connection_pop_request again, and should immediately return.
439  *
440  * If the request can't be written to the connection because it the connection
441  * has become unusable, this callback should call
442  * `fr_connection_signal_reconnect(conn)` to notify the connection API that the
443  * connection is unusable. The current request will either fail, or be
444  * re-enqueued depending on the trunk configuration.
445  *
446  * After calling #fr_connection_signal_reconnect this callback *MUST NOT*
447  * call #fr_trunk_connection_pop_request again, and should immediately return.
448  *
449  * If the protocol request data can't be written to the connection because the
450  * data is invalid or because some other error occurred, this callback should
451  * call `fr_trunk_request_signal_fail(treq)`, this callback may then continue
452  * popping/processing other requests.
453  *
454  * @param[in] el For timer management.
455  *
456  * @param[in] tconn The trunk connection to dequeue trunk
457  * requests from.
458  * @param[in] conn Connection to write the request to.
459  * Use conn->h to access the
460  * connection handle or file descriptor.
461  * @param[in] uctx User context data passed to #fr_trunk_alloc.
462  */
463 typedef void (*fr_trunk_request_mux_t)(fr_event_list_t *el,
464  fr_trunk_connection_t *tconn, fr_connection_t *conn, void *uctx);

467  *
468  * This callback should either:
469  *
470  * - If an underlying I/O library is used, request complete responses from
471  * the I/O library, and match the responses with a treq (trunk request)
472  * using a tracking structure associated with the #fr_connection_t or uctx.
473  * - If no underlying I/O library is used, read responses from the #fr_connection_t,
474  * decode those responses, and match those responses with a treq using a tracking
475  * structure associated with the #fr_connection_t or uctx.
476  *

487  * used for reporting failures at an I/O layer level not failures of queries or
488  * external services.
489  *

491  * @param[in] conn Connection to read the request from.

498 /** Inform a remote service like a datastore that a request should be cancelled
499  *
500  * This callback will be called any time there are one or more requests to be
501  * cancelled and a #fr_connection_t is writable, or as soon as a request is
502  * cancelled if `always_writable == true`.
503  *
504  * For efficiency, this callback should call #fr_trunk_connection_pop_cancellation
505  * multiple times, and process all outstanding cancellation requests.
506  *

517  * remove the entry from the tracking tree and call

520  * @param[in] tconn The trunk connection used to dequeue

529 /** Remove an outstanding "sent" request from a tracking/matching structure
530  *
531  * If the treq (trunk request) is in the FR_TRUNK_REQUEST_STATE_PARTIAL or
532  * FR_TRUNK_REQUEST_STATE_SENT states, this callback will be called prior
533  * to moving the treq to a new connection, requeueing the tre or freeing
534  * the treq.
535  *
536  * The treq, and any associated resources, should be
537  * removed from the the matching structure associated with the
538  * #fr_connection_t or uctx.
539  *
540  * Which resources should be freed depends on the cancellation reason:
541  *
542  * - FR_TRUNK_CANCEL_REASON_REQUEUE - If an encoded request can be
543  * reused, then it should be kept, otherwise it should be freed.
544  * Any resources like ID allocations bound to that request should
545  * also be freed.
546  * #fr_trunk_request_conn_release_t callback will not be called in this
547  * instance and cannot be used as an alternative.
548  * - FR_TRUNK_CANCEL_REASON_MOVE - If an encoded request can be reused
549  * it should be kept. The trunk mux callback should be aware that
550  * an encoded request may already be associated with a preq and use
551  * that instead of re-encoding the preq.
552  * If the encoded request cannot be reused it should be freed, and
553  * any fields in the preq that were modified during the last mux call
554  * (other than perhaps counters) should be reset to their initial values.
555  * Alternatively the #fr_trunk_request_conn_release_t callback can be used for
556  * the same purpose, as that will be called before the request is moved.
557  * - FR_TRUNK_CANCEL_REASON_SIGNAL - The encoded request and any I/O library
558  * request handled may be freed though that may (optionally) be left to
559  * another callback like #fr_trunk_request_conn_release_t, as that will be
560  * called as the treq is removed from the conn.
561  * Note that the #fr_trunk_request_complete_t and
562  * #fr_trunk_request_fail_t callbacks will not be called in this
563  * instance.
564  *
565  * After this callback is complete one of several actions will be taken:
566  *
567  * - If the cancellation reason was FR_TRUNK_CANCEL_REASON_REQUEUE the
568  * treq will be placed back into the pending list of the connection it
569  * was previously associated with.
570  * - If the cancellation reason was FR_TRUNK_CANCEL_REASON_MOVE, the treq
571  * will move to the unassigned state, and then either be placed in the
572  * trunk backlog, or immediately enqueued on another trunk connection.
573  * - If the reason was FR_TRUNK_CANCEL_SIGNAL
574  * - ...and a request_cancel_mux callback was provided, the
575  * the request_cancel_mux callback will be called when the connection
576  * is next writable (or immediately if `always_writable == true`) and
577  * the request_cancel_mux callback will send an explicit cancellation
578  * request to terminate any outstanding queries on remote datastores.
579  * - ...and no request_cancel_mux callback was provided, the
580  * treq will enter the unassigned state and then be freed.
581  *
582  * @note FR_TRUNK_CANCEL_REASON_MOVE will only be set if the underlying
583  * connection is bad. A 'sent' treq will never be moved due to load
584  * balancing.
585  *
586  * @note There is no need to signal request state changes in the cancellation
587  * function. The trunk will move the request into the correct state.
588  * This callback is only to allow the API client to cleanup the preq in
589  * preparation for the cancellation event.
590  *
591  * @note Cancellation requests to a remote datastore should not be made
592  * here. If that is required, a cancel_mux function should be provided.
593  *
594  * @param[in] conn to remove request from.
595  * @param[in] preq_to_reset Preq to reset.
596  * @param[in] reason Why the request was cancelled.
597  * @param[in] uctx User context data passed to #fr_trunk_alloc.
598  */
599 typedef void (*fr_trunk_request_cancel_t)(fr_connection_t *conn, void *preq_to_reset,
600  fr_trunk_cancel_reason_t reason, void *uctx);

771  *
772  * - writable - The connection is writable (the muxer will be called).
773  * - readable - The connection is readable (the demuxer will be called).

779  * assigned new requests:
780  *
781  * - inactive - The connection cannot accept any new requests. Either due to
782  * congestion or some other administrative reason.
783  * - active - The connection can, once again, accept new requests.

785  * Note: In normal operation a connection will automatically transition between
786  * the active and inactive states if conf->max_req_per_conn is specified and the
