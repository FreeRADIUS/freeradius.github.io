28  * Overview
29  * ========
30  *
31  * Read and understand this http://redis.io/topics/cluster-spec first, else the text below
32  * will not be useful.
33  *
34  * Using the cluster's public API
35  * ------------------------------
36  *
37  * The two functions used at runtime to issue commands are #fr_redis_cluster_state_init
38  * and #fr_redis_cluster_state_next.
39  *
40  * #fr_redis_cluster_state_init initialises the structure we use to track which node
41  * we're communicating with, and uses a key (string) to determine which node we should
42  * try and contact first.
43  *
44  * #fr_redis_cluster_state_next examines the result of the last command, and either
45  * gets a new connection, or errors out.
46  *
47  * In both cases the connection should not be released by the caller, only by
48  * #fr_redis_cluster_state_next.
49  *
50  * Below is the sequence of calls required to use the cluster:
51  *
52  * 1. During initialization allocate a cluster configuration structure with
53  * #fr_redis_cluster_alloc. This holds the cluster configuration and state.
54  * 2. Declare a #fr_redis_cluster_state_t in the function that needs to issue
55  * commands against the cluster.
56  * 3. Call #fr_redis_cluster_state_init with relevant arguments including a pointer
57  * to the #fr_redis_cluster_state_t struct, and a key/key_len used for initial
58  * lookup. For most commands the key is the value of the second argument.
59  * #fr_redis_cluster_state_init will then and reserve/pass back a connection for
60  * the pool associated with the node associated with the key.
61  * 4. Use the connection that was passed back, to issue a Redis command against.
62  * 5. Use #fr_redis_command_status to translate the result of the command into
63  * a #fr_redis_rcode_t value.
64  * 6. Call #fr_redis_cluster_state_next with relevant arguments including a pointer
65  * to the #fr_redis_cluster_state_t struct and the #fr_redis_rcode_t value for the
66  * last command.
67  * 7. If #fr_redis_cluster_state_next returns 0, repeat steps 4-7. Otherwise
68  * stop and analyse the return value.
69  *
70  * See #fr_redis_cluster_state_init for example code.
71  *
72  * Structures
73  * ----------
74  *
75  * This code maintains a series structures for efficient lookup and lockless operations.
76  *
77  * The important ones are:
78  * - An array of #fr_redis_cluster_node_t. These are pre-allocated on startup and are
79  * never added to, or removed from.
80  * - An #fr_fifo_t. This contains the queue of nodes that may be re-used.
81  * - An #rbtree_t. This contains a tree of nodes which are active. The tree is built on IP
82  * address and port.
83  *
84  * Each #fr_redis_cluster_node_t contains a master ID, and an array of slave IDs. The IDs are array
85  * indexes in the fr_redis_cluster_t.node array. We use 8bit unsigned integers instead of
86  * pointers to save space. Using pointers, the node[] array would need 784K, using IDs
87  * it uses 112K. Still not light on memory, but a bit more acceptable.
88  * Currently the key_slot array is shadowed by key_slot_pending, used to stage new key_slot
89  * mappings. This doubles the memory used. We may want to consider allocating key_slot_pending
90  * only during remappings and freeing it after.
91  *
92  * Mapping/Remapping the cluster
93  * -----------------------------
94  *
95  * On startup, and during cluster operation, a remap may be performed. A remap involves
96  * the following steps:
97  *
98  * 1. Executing the Redis 'cluster slots' command.
99  * 2. Validating the result of this command. We need to do extensive validation to
100  * avoid SEGV on invalid data, due to the way libhiredis presents the result.
101  * 3. Determining the intersection between nodes described in the result, and those already
102  * in our #rbtree_t.
103  * 4. Connecting to nodes that were in the result, but not in the tree.
104  * Note: If we can't connect to any of the masters, we count the map as invalid, roll
105  * back any newly connected nodes, and error out. Slave failure is OK.
106  * 5. Mapping keyslot ranges to nodes in the key_slot_pending array.
107  * 6. Verifying there are no holes in the ranges (if there are, we roll back and error out).
108  * 7. Applying the new keyslot ranges.
109  * 8. Removing nodes no longer used by the key slots, and adding them back to the free
110  * nodes queue.
111  *
112  * #cluster_map_get and #cluster_map_apply, perform the operations described
113  * above. The get function, issues the 'cluster slots' command and performs validation, the
114  * apply function processes and applys the map.
115  *
116  * Failing to apply a map is not a fatal error at runtime, and is only fatal on startup if
117  * pool.start > 0.
118  *
119  * The cluster client can continue to operate, albeit inefficiently, with a stale cluster map
120  * by following '-ASK' and '-MOVE' redirects.
121  *
122  * Remaps are limited to one per second. If any operation sets the remap_needed flag, or
123  * attempts a remap directly, the remap may be skipped if one occurred recently.
124  *
125  *
126  * Processing '-ASK' and '-MOVE' redirects
127  * ---------------------------------------
128  *
129  * The code treats '-ASK' (temporary redirect) and '-MOVE' (permanent redirect) responses
130  * similarly. If the node is known, then a connection is reserved from its pool, if the node
131  * is not known, a new pool is established, and a connection reserved.
132  *
133  * The difference between '-ASK' and '-MOVE' is that '-MOVE' attempts a cluster remap before
134  * following the redirect.
135  *
136  * The data from '-MOVE' responses, is not used to alter the cluster map. That is only done
137  * on successful remap.
138  *
139  *
140  * Processing '-TRYAGAIN'
141  * ----------------------
142  *
143  * If the cluster is in a state of flux, a node may return '-TRYAGAIN' to indicated that we
144  * should attempt the operation again. The cluster spec says we should attempt the operation
145  * after some time. This time is configurable.
146  *
147  */

274 /** Resolve key to key slot
275  *
276  * Identical to the example implementation, except it uses memchr which will
277  * be faster, and isn't so needlessly complex.
278  *
279  * @param[in] key to resolve.
280  * @param[in] key_len length of key.
281  * @return key slot index for the key.
282  */
283 static uint16_t cluster_key_hash(uint8_t const *key, size_t key_len)
284 {
285  uint8_t *p, *q;
286 
287  p = memchr(key, '{', key_len);
288  if (!p) {
289  all:
290  return fr_crc16_xmodem(key, key_len) & (KEY_SLOTS - 1);
291  }
292 
293  q = memchr(p, '}', key_len - (p - key)); /* look for } after { */
294  if (!q || (q == p + 1)) goto all; /* no } or {}, hash everything */
295 
296  p++; /* skip '{' */
297 
298  return fr_crc16_xmodem(p, q - p) & (KEY_SLOTS - 1); /* hash stuff between { and } */
299 }

346 /** Establish a connection to a cluster node
347  *
348  * @note Must be called with the cluster mutex locked.
349  * @note Configuration to use for the connection must be set in node->pending_addr, not node->cluster->conf.
350  *
351  * @param[in] cluster to search in.
352  * @param[in] node config.
353  * @return
354  * - FR_REDIS_CLUSTER_RCODE_SUCCESS on success.
355  * - FR_REDIS_CLUSTER_RCODE_FAILED if the operation failed.
356  */
357 static fr_redis_cluster_rcode_t cluster_node_connect(fr_redis_cluster_t *cluster, fr_redis_cluster_node_t *node)
358 {
359  char const *p;
360 
361  fr_assert(node->pending_addr.inet.dst_ipaddr.af);
362 
363  /*
364  * Write out the IP address and Port in string form
365  */
366  p = inet_ntop(node->pending_addr.inet.dst_ipaddr.af, &node->pending_addr.inet.dst_ipaddr.addr,
367  node->name, sizeof(node->name));
368  if (!fr_cond_assert(p)) return FR_REDIS_CLUSTER_RCODE_FAILED;
369 
370  /*
371  * Node has never been used before, needs a pool allocated for it.
372  */
373  if (!node->pool) {
374  char buffer[256];
375  fr_pair_t *args;
376  CONF_SECTION *pool;
377 
378  snprintf(buffer, sizeof(buffer), "%s [%i]", cluster->log_prefix, node->id);
379 
380  pool = cf_section_find(cluster->module, "pool", NULL);
381  /*
382  * Dup so we can re-parse, and have unique CONF_DATA
383  */
384  node->pool_cs = cf_section_dup(cluster, NULL, pool, "pool", NULL, true);
385  node->addr = node->pending_addr;
386  node->pool = fr_pool_init(cluster, node->pool_cs, node,
387  fr_redis_cluster_conn_create, NULL, buffer);
388  if (!node->pool) {
389  error:
390  TALLOC_FREE(node->pool_cs);
391  TALLOC_FREE(node->pool);
392  return FR_REDIS_CLUSTER_RCODE_FAILED;
393  }
394  fr_pool_reconnect_func(node->pool, _cluster_node_conf_apply);
395 
396  if (trigger_enabled() && cluster->triggers_enabled) {
397  args = trigger_args_afrom_server(node->pool, node->name, node->addr.inet.dst_port);
398  if (!args) goto error;
399 
400  if (cluster->trigger_args) MEM(fr_pair_list_copy(cluster, &args, &cluster->trigger_args) >= 0);
401 
402  fr_pool_enable_triggers(node->pool, node->cluster->trigger_prefix, args);
403 
404  fr_pair_list_free(&args);
405  }
406 
407  if (fr_pool_start(node->pool) < 0) goto error;
408 
409  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
410  }
411 
412  /*
413  * Apply the new config to the possibly live pool
414  */
415  if (fr_pool_reconnect(node->pool, NULL) < 0) goto error;
416 
417  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
418 }

420 /** Parse a -MOVED or -ASK redirect
421  *
422  * Converts the body of the -MOVED or -ASK error into an IPv4/6 address and port.
423  *
424  * @param[out] key_slot value extracted from redirect string (may be NULL).
425  * @param[out] node_addr Redis node ipaddr and port extracted from redirect string.
426  * @param[in] redirect to process.
427  * @return
428  * - FR_REDIS_CLUSTER_RCODE_SUCCESS on success.
429  * - FR_REDIS_CLUSTER_RCODE_BAD_INPUT if the server returned an invalid redirect.
430  */
431 static fr_redis_cluster_rcode_t cluster_node_conf_from_redirect(uint16_t *key_slot, fr_socket_t *node_addr,
432  redisReply *redirect)
433 {
434  char *p, *q;
435  unsigned long key;
436  uint16_t port;
437  fr_ipaddr_t ipaddr;
438 
439  if (!redirect || (redirect->type != REDIS_REPLY_ERROR)) {
440  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
441  }
442 
443  p = redirect->str;
444  if (strncmp(REDIS_ERROR_MOVED_STR, redirect->str, sizeof(REDIS_ERROR_MOVED_STR) - 1) == 0) {
445  q = p + sizeof(REDIS_ERROR_MOVED_STR); /* not a typo, skip space too */
446  } else if (strncmp(REDIS_ERROR_ASK_STR, redirect->str, sizeof(REDIS_ERROR_ASK_STR) - 1) == 0) {
447  q = p + sizeof(REDIS_ERROR_ASK_STR); /* not a typo, skip space too */
448  } else {
449  fr_strerror_printf("No '-MOVED' or '-ASK' log_prefix");
450  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
451  }
452  if ((size_t)(q - p) >= (size_t)redirect->len) {
453  fr_strerror_printf("Truncated");
454  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
455  }
456  p = q;
457  key = strtoul(p, &q, 10);
458  if (key > KEY_SLOTS) {
459  fr_strerror_printf("Key %lu outside of redis slot range", key);
460  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
461  }
462  p = q;
463 
464  if (*p != ' ') {
465  fr_strerror_printf("Missing key/host separator");
466  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
467  }
468  p++; /* Skip the ' ' */
469 
470  if (fr_inet_pton_port(&ipaddr, &port, p, redirect->len - (p - redirect->str), AF_UNSPEC, false, true) < 0) {
471  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
472  }
473  fr_assert(ipaddr.af);
474 
475  if (key_slot) *key_slot = key;
476  if (node_addr) {
477  node_addr->inet.dst_ipaddr = ipaddr;
478  node_addr->inet.dst_port = port;
479  }
480 
481  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
482 }

526

531  bool master[UINT8_MAX]; // Master nodes.

535  int _ret; \

537  fr_assert(_ret == 0);\
538  _addr.inet.dst_port = _map->element[1]->integer; \
539 } while (0)
540 #else
541 # define SET_ADDR(_addr, _map) \
542 do { \
543  fr_inet_pton(&_addr.inet.dst_ipaddr, _map->element[0]->str, _map->element[0]->len, AF_UNSPEC, false, true);\
544  _addr.inet.dst_port = _map->element[1]->integer; \
545 } while (0)
546 #endif

549 do { \
550  (_node)->is_active = false; \

567  memset(&rollback, 0, sizeof(rollback));
568  memset(active, 0, sizeof(active));
569  memset(master, 0, sizeof(master));
570 
571  cluster->remapping = true;
572 
573  /*
574  * Must be cleared with the mutex held
575  */
576  memset(&cluster->key_slot_pending, 0, sizeof(cluster->key_slot_pending));
577 
578  /*
579  * Insert new nodes and markup the keyslot indexes
580  * in our temporary keyslot_array.
581  *
582  * A map consists of an array with the following indexes:
583  * [0] -> key_slot_start
584  * [1] -> key_slot_end
585  * [2] -> master_node
586  * [3..n] -> slave_node(s)
587  */
588  for (i = 0; i < reply->elements; i++) {
589  size_t j;
590  long long int k;
591  int slaves = 0;
592  fr_redis_cluster_node_t *found, *spare;
593  fr_redis_cluster_node_t find;
594  fr_redis_cluster_key_slot_t tmpl_slot;
595  redisReply *map = reply->element[i];
596 
597  memset(&tmpl_slot, 0, sizeof(tmpl_slot));
598 
599  SET_ADDR(find.addr, map->element[2]);
600  found = rbtree_finddata(cluster->used_nodes, &find);
601  if (found) {
602  active[found->id] = true;
603  goto reuse_master_node;
604  }
605 
606  /*
607  * Process the master
608  *
609  * A master node consists of any array with the following indexes:
610  * [0] -> node ip (as string)
611  * [1] -> node port
612  */
613  spare = fr_fifo_peek(cluster->free_nodes);
614  if (!spare) {
615  out_of_nodes:
616  fr_strerror_printf("Reached maximum connected nodes");
617  rcode = FR_REDIS_CLUSTER_RCODE_FAILED;
618  error:
619  cluster->remapping = false;
620  cluster->last_updated = time(NULL);
621  /* Re-insert new nodes back into the free_nodes queue */
622  for (i = 0; i < r; i++) SET_INACTIVE(&cluster->node[rollback[i]]);
623  return rcode;
624  }
625 
626  spare->pending_addr = find.addr;
627  rcode = cluster_node_connect(cluster, spare);
628  if (rcode < 0) goto error;
629 
630  /*
631  * Check to see if the node we just configured
632  * already exists in the tree. If it does we
633  * use that, else we add it to the array of
634  * nodes to rollback on failure.
635  */
636  SET_ACTIVE(spare);
637  found = spare;
638 
639  reuse_master_node:
640  tmpl_slot.master = found->id;
641  master[found->id] = true; /* Mark this node as a master */
642 
643  /*
644  * Process the slaves
645  *
646  * A slave node consists of any array with the following indexes:
647  * [0] -> node ip (as string)
648  * [1] -> node port
649  */
650  for (j = 3; (j < map->elements); j++) {
651  SET_ADDR(find.addr, map->element[j]);
652  found = rbtree_finddata(cluster->used_nodes, &find);
653  if (found) {
654  active[found->id] = true;
655  goto next;
656  }
657 
658  spare = fr_fifo_peek(cluster->free_nodes);
659  if (!spare) goto out_of_nodes;
660 
661  spare->pending_addr = find.addr;
662  if (cluster_node_connect(cluster, spare) < 0) continue; /* Slave failure is non-fatal */
663 
664  SET_ACTIVE(spare);
665  found = spare;
666 
667  next:
668  tmpl_slot.slave[slaves++] = found->id;
669 
670  /* Hit the maximum number of slaves we allow */
671  if (slaves >= MAX_SLAVES) break;
672  }
673  tmpl_slot.slave_num = slaves;
674 
675  /*
676  * Copy our tmpl key slot to each of the key slots
677  * specified by the range for this map.
678  */
679  for (k = map->element[0]->integer; k <= map->element[1]->integer; k++) {
680  memcpy(&cluster->key_slot_pending[k], &tmpl_slot,
681  sizeof(*(cluster->key_slot_pending)));
682  }
683  }
684 
685  /*
686  * Check for holes in the pending_addr key_slot array
687  *
688  * The cluster specification says that upon
689  * detecting a 'NULL' key_slot we should
690  * check again to see if the cluster error has
691  * been resolved, but seeing as we're in the
692  * middle of updating the cluster from very
693  * recent output of 'cluster slots' it's best to
694  * error out.
695  */
696  for (i = 0; i < KEY_SLOTS; i++) {
697  if (cluster->key_slot_pending[i].master == 0) {
698  fr_strerror_printf("Cluster is misconfigured, no node assigned for key %zu", i);
699  rcode = FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
700  goto error;
701  }
702  }
703 
704  /*
705  * We have connections/pools for all the nodes in
706  * the new map, apply it to the live cluster.
707  *
708  * Other workers may be using the key slot table,
709  * but that's ok. Nodes and pools are never freed,
710  * so the worst that will happen, is they'll hit
711  * the wrong node for the key, and get redirected.
712  */
713  memcpy(&cluster->key_slot, &cluster->key_slot_pending, sizeof(cluster->key_slot));
714 
715  /*
716  * Anything not in the active set of nodes gets
717  * added back into the queue, to be re-used.
718  *
719  * We start at 1, as node 0 is reserved.
720  */
721  for (i = 1; i < cluster->conf->max_nodes; i++) {
722 #ifndef NDEBUG
723  fr_redis_cluster_node_t *found;
724 
725  if (cluster->node[i].is_active) {
726  /* Sanity check for duplicates that are active */
727  found = rbtree_finddata(cluster->used_nodes, &cluster->node[i]);
728  fr_assert(found);
729  fr_assert(found->is_active);
730  fr_assert(found->id == i);
731  }
732 #endif
733 
734  if (!active[i] && cluster->node[i].is_active) {
735  SET_INACTIVE(&cluster->node[i]); /* Sets is_master = false */
736 
737  /*
738  * Only change the masters once we've successfully
739  * remapped the cluster.
740  */
741  } else if (master[i]) {
742  cluster->node[i].is_master = true;
743  }
744  }
745 
746  cluster->remapping = false;
747  cluster->last_updated = time(NULL);
748 
749  /*
750  * Sanity checks
751  */
752  fr_assert(((talloc_array_length(cluster->node) - 1) - rbtree_num_elements(cluster->used_nodes)) ==
753  fr_fifo_num_elements(cluster->free_nodes));
754 
755  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
756 }

758 /** Validate a cluster map node entry
759  *
760  * @note Errors may be retrieved with fr_strerror().
761  * @note In a separate function, as it's called for both master and slave nodes.
762  *
763  * @param[in] node we're validating.
764  * @param[in] map_idx we're processing.
765  * @param[in] node_idx we're processing.
766  * @return
767  * - FR_REDIS_CLUSTER_RCODE_SUCCESS on success.
768  * - FR_REDIS_CLUSTER_RCODE_BAD_INPUT on validation failure (bad data returned from Redis).
769  */
770 static int cluster_map_node_validate(redisReply *node, int map_idx, int node_idx)
771 {
772  fr_ipaddr_t ipaddr;
773 
774  if (node->type != REDIS_REPLY_ARRAY) {
775  fr_strerror_printf("Cluster map %i node %i is wrong type, expected array got %s",
776  map_idx, node_idx,
777  fr_table_str_by_value(redis_reply_types, node->element[1]->type, "<UNKNOWN>"));
778  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
779  }
780 
781  /*
782  * As per the redis docs: https://redis.io/commands/cluster-slots
783  *
784  * Newer versions of Redis Cluster will output, for each Redis instance,
785  * not just the IP and port, but also the node ID as third element of the
786  * array. In future versions there could be more elements describing the
787  * node better. In general a client implementation should just rely on
788  * the fact that certain parameters are at fixed positions as specified,
789  * but more parameters may follow and should be ignored.
790  * Similarly a client library should try if possible to cope with the fact
791  * that older versions may just have the IP and port parameter.
792  */
793  if (node->elements < 2) {
794  fr_strerror_printf("Cluster map %i node %i has incorrect number of elements, expected at least "
795  "2 got %zu", map_idx, node_idx, node->elements);
796  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
797  }
798 
799  if (node->element[0]->type != REDIS_REPLY_STRING) {
800  fr_strerror_printf("Cluster map %i node %i ip address is wrong type, expected string got %s",
801  map_idx, node_idx,
802  fr_table_str_by_value(redis_reply_types, node->element[0]->type, "<UNKNOWN>"));
803  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
804  }
805 
806  if (fr_inet_pton(&ipaddr, node->element[0]->str, node->element[0]->len, AF_UNSPEC, false, true) < 0) {
807  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
808  }
809 
810  if (node->element[1]->type != REDIS_REPLY_INTEGER) {
811  fr_strerror_printf("Cluster map %i node %i port is wrong type, expected integer got %s",
812  map_idx, node_idx,
813  fr_table_str_by_value(redis_reply_types, node->element[1]->type, "<UNKNOWN>"));
814  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
815  }
816 
817  if (node->element[1]->integer < 0) {
818  fr_strerror_printf("Cluster map %i node %i port is too low, expected >= 0 got %lli",
819  map_idx, node_idx, node->element[1]->integer);
820  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
821  }
822 
823  if (node->element[1]->integer > UINT16_MAX) {
824  fr_strerror_printf("Cluster map %i node %i port is too high, expected <= " STRINGIFY(UINT16_MAX)" "
825  "got %lli", map_idx, node_idx, node->element[1]->integer);
826  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
827  }
828 
829  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
830 }

832 /** Learn a new cluster layout by querying the node that issued the -MOVE
833  *
834  * Also validates the response from the Redis cluster, so we can be sure that
835  * it's well formed, before doing more expensive operations.
836  *
837  * @note Errors may be retrieved with fr_strerror().
838  *
839  * @param[out] out Where to write cluster map.
840  * @param[in] conn to use for learning the new cluster map.
841  * @return
842  * - FR_REDIS_CLUSTER_RCODE_IGNORED if 'cluster slots' returned an error (indicating clustering not supported).
843  * - FR_REDIS_CLUSTER_RCODE_SUCCESS on success.
844  * - FR_REDIS_CLUSTER_RCODE_FAILED if issuing the command resulted in an error.
845  * - FR_REDIS_CLUSTER_RCODE_NO_CONNECTION connection failure.
846  * - FR_REDIS_CLUSTER_RCODE_BAD_INPUT on validation failure (bad data returned from Redis).
847  */
848 static fr_redis_cluster_rcode_t cluster_map_get(redisReply **out, fr_redis_conn_t *conn)
849 {
850  redisReply *reply;
851  size_t i = 0;
852 
853  *out = NULL;
854 
855  reply = redisCommand(conn->handle, "cluster slots");
856  switch (fr_redis_command_status(conn, reply)) {
857  case REDIS_RCODE_RECONNECT:
858  fr_redis_reply_free(&reply);
859  fr_strerror_printf("No connections available");
860  return FR_REDIS_CLUSTER_RCODE_NO_CONNECTION;
861 
862  case REDIS_RCODE_ERROR:
863  default:
864  if (reply && reply->type == REDIS_REPLY_ERROR) {
865  fr_strerror_printf("%.*s", (int)reply->len, reply->str);
866  fr_redis_reply_free(&reply);
867  return FR_REDIS_CLUSTER_RCODE_IGNORED;
868  }
869  fr_strerror_printf("Unknown client error");
870  return FR_REDIS_CLUSTER_RCODE_FAILED;
871 
872  case REDIS_RCODE_SUCCESS:
873  break;
874  }
875 
876  if (reply->type != REDIS_REPLY_ARRAY) {
877  fr_strerror_printf("Bad response to \"cluster slots\" command, expected array got %s",
878  fr_table_str_by_value(redis_reply_types, reply->type, "<UNKNOWN>"));
879  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
880  }
881 
882  /*
883  * Clustering configured but no slots set
884  */
885  if (reply->elements == 0) {
886  fr_strerror_printf("Empty response to \"cluster slots\" command (zero length array)");
887  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
888  }
889 
890  /*
891  * Validate the complete map set before returning.
892  */
893  for (i = 0; i < reply->elements; i++) {
894  size_t j;
895  redisReply *map;
896 
897  map = reply->element[i];
898  if (map->type != REDIS_REPLY_ARRAY) {
899  fr_strerror_printf("Cluster map %zu is wrong type, expected array got %s",
900  i, fr_table_str_by_value(redis_reply_types, map->type, "<UNKNOWN>"));
901  error:
902  fr_redis_reply_free(&reply);
903  return FR_REDIS_CLUSTER_RCODE_BAD_INPUT;
904  }
905 
906  if (map->elements < 3) {
907  fr_strerror_printf("Cluster map %zu has too few elements, expected at least 3, got %zu",
908  i, map->elements);
909  goto error;
910  }
911 
912  /*
913  * Key slot start
914  */
915  if (map->element[0]->type != REDIS_REPLY_INTEGER) {
916  fr_strerror_printf("Cluster map %zu key slot start is wrong type, expected integer got %s",
917  i, fr_table_str_by_value(redis_reply_types, map->element[0]->type, "<UNKNOWN>"));
918  goto error;
919  }
920 
921  if (map->element[0]->integer < 0) {
922  fr_strerror_printf("Cluster map %zu key slot start is too low, expected >= 0 got %lli",
923  i, map->element[0]->integer);
924  goto error;
925  }
926 
927  if (map->element[0]->integer > KEY_SLOTS) {
928  fr_strerror_printf("Cluster map %zu key slot start is too high, expected <= "
929  STRINGIFY(KEY_SLOTS) " got %lli", i, map->element[0]->integer);
930  goto error;
931  }
932 
933  /*
934  * Key slot end
935  */
936  if (map->element[1]->type != REDIS_REPLY_INTEGER) {
937  fr_strerror_printf("Cluster map %zu key slot end is wrong type, expected integer got %s",
938  i, fr_table_str_by_value(redis_reply_types, map->element[1]->type, "<UNKNOWN>"));
939  goto error;
940  }
941 
942  if (map->element[1]->integer < 0) {
943  fr_strerror_printf("Cluster map %zu key slot end is too low, expected >= 0 got %lli",
944  i, map->element[1]->integer);
945  goto error;
946  }
947 
948  if (map->element[1]->integer > KEY_SLOTS) {
949  fr_strerror_printf("Cluster map %zu key slot end is too high, expected <= "
950  STRINGIFY(KEY_SLOTS) " got %lli", i, map->element[1]->integer);
951  goto error;
952  }
953 
954  if (map->element[1]->integer < map->element[0]->integer) {
955  fr_strerror_printf("Cluster map %zu key slot start/end out of order. "
956  "Start was %lli, end was %lli", i, map->element[0]->integer,
957  map->element[1]->integer);
958  goto error;
959  }
960 
961  /*
962  * Master node
963  */
964  if (cluster_map_node_validate(map->element[2], i, 0) < 0) goto error;
965 
966  /*
967  * Slave nodes
968  */
969  for (j = 3; j < map->elements; j++) {
970  if (cluster_map_node_validate(map->element[j], i, j - 2) < 0) goto error;
971  }
972  }
973  *out = reply;
974 
975  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
976 }

978 /** Perform a runtime remap of the cluster
979  *
980  * @note Errors may be retrieved with fr_strerror().
981  * @note Must be called with the cluster mutex free.
982  *
983  * @param[in] request The current request.
984  * @param[in,out] cluster to remap.
985  * @param[in] conn to use to query the cluster.
986  * @return
987  * - FR_REDIS_CLUSTER_RCODE_IGNORED if 'cluster slots' returned an error (indicating clustering not supported).
988  * - FR_REDIS_CLUSTER_RCODE_SUCCESS on success.
989  * - FR_REDIS_CLUSTER_RCODE_FAILED if issuing the 'cluster slots' command resulted in a protocol error.
990  * - FR_REDIS_CLUSTER_RCODE_NO_CONNECTION connection failure.
991  * - FR_REDIS_CLUSTER_RCODE_BAD_INPUT on validation failure (bad data returned from Redis).
992  */
993 fr_redis_cluster_rcode_t fr_redis_cluster_remap(request_t *request, fr_redis_cluster_t *cluster, fr_redis_conn_t *conn)
994 {
995  time_t now;
996  redisReply *map;
997  fr_redis_cluster_rcode_t ret;
998  size_t i, j;
999 
1000  /*
1001  * If the cluster was remapped very recently, or is being
1002  * remapped it's unlikely that it needs remapping again.
1003  */
1004  if (cluster->remapping) {
1005  in_progress:
1006  RDEBUG2("Cluster remapping in progress, ignoring remap request");
1007  return FR_REDIS_CLUSTER_RCODE_IGNORED;
1008  }
1009 
1010  now = time(NULL);
1011  if (now == cluster->last_updated) {
1012  too_soon:
1013  RWARN("Cluster was updated less than a second ago, ignoring remap request");
1014  return FR_REDIS_CLUSTER_RCODE_IGNORED;
1015  }
1016 
1017  RINFO("Initiating cluster remap");
1018 
1019  /*
1020  * Get new cluster information
1021  */
1022  ret = cluster_map_get(&map, conn);
1023  switch (ret) {
1024  case FR_REDIS_CLUSTER_RCODE_BAD_INPUT: /* Validation error */
1025  case FR_REDIS_CLUSTER_RCODE_NO_CONNECTION: /* Connection error */
1026  case FR_REDIS_CLUSTER_RCODE_FAILED: /* Error issuing command */
1027  return ret;
1028 
1029  case FR_REDIS_CLUSTER_RCODE_IGNORED: /* Clustering not enabled, or not supported */
1030  cluster->remap_needed = false;
1031  return FR_REDIS_CLUSTER_RCODE_IGNORED;
1032 
1033  case FR_REDIS_CLUSTER_RCODE_SUCCESS: /* Success */
1034  break;
1035  }
1036 
1037  /*
1038  * Print the mapping we received
1039  */
1040  RINFO("Cluster map consists of %zu key ranges", map->elements);
1041  for (i = 0; i < map->elements; i++) {
1042  redisReply *map_node = map->element[i];
1043 
1044  RINFO("%zu - keys %lli-%lli", i,
1045  map_node->element[0]->integer,
1046  map_node->element[1]->integer);
1047 
1048  RINDENT();
1049  RINFO("master: %s:%lli",
1050  map_node->element[2]->element[0]->str,
1051  map_node->element[2]->element[1]->integer);
1052  for (j = 3; j < map_node->elements; j++) {
1053  RINFO("slave%zu: %s:%lli", j - 3,
1054  map_node->element[j]->element[0]->str,
1055  map_node->element[j]->element[1]->integer);
1056  }
1057  REXDENT();
1058  }
1059 
1060  /*
1061  * Check again that the cluster isn't being
1062  * remapped, or was remapped too recently,
1063  * now we hold the mutex and the state of
1064  * those variables is synchronized.
1065  */
1066  pthread_mutex_lock(&cluster->mutex);
1067  if (cluster->remapping) {
1068  pthread_mutex_unlock(&cluster->mutex);
1069  fr_redis_reply_free(&map); /* Free the map */
1070  goto in_progress;
1071  }
1072  if (now == cluster->last_updated) {
1073  pthread_mutex_unlock(&cluster->mutex);
1074  fr_redis_reply_free(&map); /* Free the map */
1075  goto too_soon;
1076  }
1077  ret = cluster_map_apply(cluster, map);
1078  if (ret == FR_REDIS_CLUSTER_RCODE_SUCCESS) cluster->remap_needed = false; /* Change on successful remap */
1079  pthread_mutex_unlock(&cluster->mutex);
1080 
1081  fr_redis_reply_free(&map); /* Free the map */
1082  if (ret < 0) return FR_REDIS_CLUSTER_RCODE_FAILED;
1083 
1084  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
1085 }

1087 /** Retrieve or associate a node with the server indicated in the redirect
1088  *
1089  * @note Errors may be retrieved with fr_strerror().
1090  *
1091  * @param[out] out Where to write the node representing the redirect server.
1092  * @param[in] cluster to draw node from.
1093  * @param[in] reply Redis reply containing the redirect information.
1094  * @return
1095  * - FR_REDIS_CLUSTER_RCODE_SUCCESS on success.
1096  * - FR_REDIS_CLUSTER_RCODE_FAILED no more nodes available.
1097  * - FR_REDIS_CLUSTER_RCODE_NO_CONNECTION connection failure.
1098  * - FR_REDIS_CLUSTER_RCODE_BAD_INPUT on validation failure (bad data returned from Redis).
1099  */
1100 static fr_redis_cluster_rcode_t cluster_redirect(fr_redis_cluster_node_t **out, fr_redis_cluster_t *cluster, redisReply *reply)
1101 {
1102  fr_redis_cluster_node_t find, *found, *spare;
1103  fr_redis_conn_t *rconn;
1104 
1105  uint16_t key;
1106 
1107  memset(&find, 0, sizeof(find));
1108 
1109  *out = NULL;
1110 
1111  if (cluster_node_conf_from_redirect(&key, &find.addr, reply) < 0) return FR_REDIS_CLUSTER_RCODE_FAILED;
1112 
1113  pthread_mutex_lock(&cluster->mutex);
1114  /*
1115  * If we have already have a pool for the
1116  * host we were redirected to, use that.
1117  */
1118  found = rbtree_finddata(cluster->used_nodes, &find);
1119  if (found) {
1120  /* We have the new pool, don't need to hold the lock */
1121  pthread_mutex_unlock(&cluster->mutex);
1122  *out = found;
1123  return FR_REDIS_CLUSTER_RCODE_SUCCESS;
1124  }
1125 
1126  /*
1127  * Otherwise grab a free node and try and connect
1128  * it to the server we were redirected to.
1129  */
1130  spare = fr_fifo_peek(cluster->free_nodes);
1131  if (!spare) {
1132  fr_strerror_printf("Reached maximum connected nodes");
1133  pthread_mutex_unlock(&cluster->mutex);
1134  return FR_REDIS_CLUSTER_RCODE_FAILED;
1135  }
1136  spare->pending_addr = find.addr; /* Set the config to be applied */
1137  if (cluster_node_connect(cluster, spare) < 0) {
1138  pthread_mutex_unlock(&cluster->mutex);
1139  return FR_REDIS_CLUSTER_RCODE_NO_CONNECTION;
1140  }
1141  rbtree_insert(cluster->used_nodes, spare);
1142  fr_fifo_pop(cluster->free_nodes);
1143  found = spare;
1144 
1145  /* We have the new pool, don't need to hold the lock */
1146  pthread_mutex_unlock(&cluster->mutex);
1147 
1148  /*
1149  * Determine if we can establish a connection to
1150  * the new pool, to check if it's viable.
1151  */
1152  rconn = fr_pool_connection_get(found->pool, NULL);
1153  if (!rconn) {
1154  /*
1155  * To prevent repeated misconfigurations
1156  * using all free nodes, add the node
1157  * back to the spare queue if this
1158  * was the first connection attempt and
1159  * it failed.
1160  */
1161  pthread_mutex_lock(&cluster->mutex);
1162  fr_fifo_push(cluster->free_nodes, spare);
1163  pthread_mutex_unlock(&cluster->mutex);
1164 
1165  fr_strerror_printf("No connections available");
1166  return FR_REDIS_CLUSTER_RCODE_NO_CONNECTION;
1167  }

1197 /** Try to determine the health of a cluster node passively by examining its pool state
1198  *
1199  * Returns an integer value representing the likelihood that the pool is live.
1200  * Range is between 1 and 11,000.
1201  *
1202  * If a weight of 1 is returned, connections from the pool should be checked
1203  * (by pinging) before use.
1204  *
1205  * @param now The current time.
1206  * @param state of the connection pool.
1207  * @return
1208  * - 1 the pool is very likely to be bad.
1209  * - 2-11000 the pool is likely to be good, with a higher number
1210  * indicating higher probability of liveness.
1211  */
1212 static int cluster_node_pool_health(fr_time_t now, fr_pool_state_t const *state)
1213 {
1214  /*
1215  * Failed spawn recently, probably bad
1216  */
1217  if (fr_time_delta_to_msec(now - state->last_failed) < FAILED_PERIOD) return FAILED_WEIGHT;
1218 
1219  /*
1220  * Closed recently, probably bad
1221  */
1222  if (fr_time_delta_to_msec(now - state->last_closed) < CLOSED_PERIOD) return CLOSED_WEIGHT;
1223 
1224  /*
1225  * Released too long ago, don't know
1226  */
1227  if (fr_time_delta_to_msec(now - state->last_released) > RELEASED_PERIOD) return RELEASED_MIN_WEIGHT;
1228 
1229  /*
1230  * Released not long ago, might be ok.
1231  */
1232  return RELEASED_MIN_WEIGHT + (RELEASED_PERIOD - fr_time_delta_to_msec(now - state->last_released));
1233 }

1274 /** Attempt to find a live pool in the cluster
1275  *
1276  * The intent here is to find pools/nodes where a connection was released the shortest
1277  * time ago. Having a connection be released (vs closed) indicates that the pool is live.
1278  *
1279  * We don't want to have all workers try and grab a connection to this node however, as it
1280  * may still be dead (we don't know).
1281  *
1282  * So we use an inverse transform sample, to weight the nodes, based on time between now
1283  * and when the connection was released. Connections released closest to the current
1284  * time are given a higher weighting.
1285  *
1286  * Weight range is between 1 - 11,000.
1287  *
1288  * - If released > 10.0 seconds ago,information is not valid, weight 500.
1289  * - If closed < 10.0 seconds ago, it's a bad pool, weight 1.
1290  * - If spawn failed < 10.0 seconds ago, it's a bad pool, weight 1.
1291  * - If a connection was released 0.0 seconds ago, weight 11,000.
1292  * - If a connection was released 10.0 seconds ago, weight 1000.
1293  *
1294  * Using the above algorithm we use the experience of other workers using the cluster to
1295  * inform our alternative node selection.
1296  *
1297  * Suggestions on improving live node selection appreciated.
1298  *
1299  * Inverse transform sampling based roughly on the solution from this post:
1300  * http://stackoverflow.com/questions/17250568/randomly-choosing-from-a-list-with-weighted-probabilities
1301  *
1302  * Wikipedia page here:
1303  * https://en.wikipedia.org/wiki/Inverse_transform_sampling
1304  *
1305  * @note Must be called with the cluster mutex free.
1306  *
1307  * @param[out] live_node we found.
1308  * @param[out] live_conn to that node.
1309  * @param[in] request The current request (used for logging).
1310  * @param[in] cluster to search for live pools in.
1311  * @param[in] skip this node (it's bad).
1312  * @return 0 (iterates over the whole tree).
1313  */
1314 static int cluster_node_find_live(fr_redis_cluster_node_t **live_node, fr_redis_conn_t **live_conn,
1315  request_t *request, fr_redis_cluster_t *cluster, fr_redis_cluster_node_t *skip)
1316 {
1317  uint32_t i;
1318 
1319  cluster_nodes_live_t *live;
1320  fr_time_t now;
1321 
1322  RDEBUG2("Searching for live cluster nodes");
1323 
1324  if (rbtree_num_elements(cluster->used_nodes) == 1) {
1325  no_alts:
1326  RERROR("No alternative nodes available");
1327  return -1;
1328  }
1329 
1330  live = talloc_zero(NULL, cluster_nodes_live_t); /* Too big for stack */
1331  live->skip = skip->id;
1332 
1333  pthread_mutex_lock(&cluster->mutex);
1334  rbtree_walk(cluster->used_nodes, RBTREE_IN_ORDER, _cluster_pool_walk, live);
1335  pthread_mutex_unlock(&cluster->mutex);
1336 
1337  fr_assert(live->next); /* There should be at least one */
1338  if (live->next == 1) goto no_alts; /* Weird, but conceivable */
1339 
1340  now = fr_time();
1341 
1342  /*
1343  * Weighted random selection
1344  */
1345  for (i = 0; (i < cluster->conf->max_alt) && live->next; i++) {
1346  fr_redis_conn_t *conn;
1347  fr_redis_cluster_node_t *node;
1348  uint8_t j;
1349  int first, last, pivot; /* Must be signed for BS */
1350  unsigned int find, cumulative = 0;
1351 
1352  RDEBUG3("(Re)assigning node weights:");
1353  RINDENT();
1354  for (j = 0; j < live->next; j++) {
1355  int weight;
1356 
1357  weight = cluster_node_pool_health(now, live->node[j].pool_state);
1358  RDEBUG3("Node %i weight: %i", live->node[j].id, weight);
1359  live->node[j].cumulative = (cumulative += weight);
1360  }
1361  REXDENT();
1362 
1363  /*
1364  * Select a node at random
1365  */
1366  find = (fr_rand() & (cumulative - 1)); /* Between 1 and total */
1367  first = 0;
1368  last = live->next - 1;
1369  pivot = (first + last) / 2;
1370 
1371  while (first <= last) {
1372  if (live->node[pivot].cumulative < find) {
1373  first = pivot + 1;
1374  } else if (live->node[pivot].cumulative == find) {
1375  break;
1376  } else {
1377  last = pivot - 1;
1378  }
1379  pivot = (first + last) / 2;
1380  }
1381  /*
1382  * Round up...
1383  */
1384  if (first > last) pivot = last + 1;
1385 
1386  /*
1387  * Resolve the index to the actual node. We use IDs
1388  * to save memory...
1389  */
1390  node = &cluster->node[live->node[pivot].id];
1391  fr_assert(live->node[pivot].id == node->id);
1392 
1393  RDEBUG2("Selected node %i (using random value %i)", node->id, find);
1394  conn = fr_pool_connection_get(node->pool, request);
1395  if (!conn) {
1396  RERROR("No connections available to node %i %s:%i", node->id,
1397  node->name, node->addr.inet.dst_port);
1398  next:
1399  /*
1400  * Remove the node we just discovered was bad
1401  * out of the set of nodes we're selecting over.
1402  */
1403  if (pivot == live->next) {
1404  live->next--;
1405  continue;
1406  }
1407  memcpy(&live->node[pivot], &live->node[live->next - 1], sizeof(live->node[pivot]));
1408  live->next--;
1409  continue;
1410  }
1411 
1412  /*
1413  * PING! PONG?
1414  */
1415  switch (cluster_node_ping(request, node, conn)) {
1416  case FR_REDIS_CLUSTER_RCODE_SUCCESS:
1417  break;
1418 
1419  case FR_REDIS_CLUSTER_RCODE_NO_CONNECTION:
1420  fr_pool_connection_close(node->pool, request, conn);
1421  goto next;
1422 
1423  default:
1424  fr_pool_connection_release(node->pool, request, conn);
1425  goto next;
1426  }
1427 
1428  *live_node = node;
1429  *live_conn = conn;
1430  talloc_free(live);
1431 
1432  return 0;
1433  }
1434 
1435  RERROR("Hit max alt limit %i, and no live connections found", cluster->conf->max_alt);
1436  talloc_free(live);
1437 
1438  return -1;
1439 }

1716

1792  *
1793  * - May trigger a cluster remap on receiving a #REDIS_RCODE_MOVE status.
1794  * - May perform a temporary redirect on receiving a #REDIS_RCODE_ASK status.
1795  * - May reserve a new connection on receiving a #REDIS_RCODE_RECONNECT status.
1796  *
1797  * If a remap is in progress, has ocurred within the last second, has recently failed,
1798  * or fails, the '-MOVE' will be treated as a temporary redirect (-ASK).
1799  *
1800  * This allows the server to be more responsive during remaps, as unless the worker has been
1801  * redirected to a node we don't currently have a pool for, it can grab a connection for the
1802  * node it was redirected to, and continue.

1805  * A new connection to attempt command on will be provided via conn.

1808  * in all other cases, the caller is responsible for freeing the reply.

1811  * retries, and limit redirects.

1813  * connection in the new pool the key points to.

1821  * - REDIS_RCODE_ERROR - on failure or command error.
1822  * - REDIS_RCODE_RECONNECT - when no additional connections available.

1988  */
