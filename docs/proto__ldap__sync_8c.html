◆ proto_ldap_socket_recv()

static int proto_ldap_socket_recv ( rad_listen_t *  listen)
static

De-multiplex incoming LDAP Messages.

This callback should be called when the LDAP socket is readable. It drains any outstanding LDAPMessages using sync_demux. sync_demux in-turn calls one of the following callbacks:

  • _proto_ldap_entry We received a notification an entry was added, deleted, or modified.
  • _proto_ldap_cookie_store We received a new cookie value.
  • _proto_ldap_present The server wants to perform a refresh present phase (which we don't allow).
  • _proto_ldap_refresh_required The server wants us to download all content specified by our sync/search.

These callbacks may result in requests being enqueued or syncs restarted.

Note
We do not currently make enqueuing the cookie requests dependent on all previous LDAP entries being processed, so we may miss updates in some circumstances. This needs to be fixed, but is waiting on v4.0.0 re-architecture.
Parameters
[in]listenencapsulating the libldap socket.
Returns
  • 1 on success.
  • 0 on failure.
